1. Introdução

O projeto teve como objetivo desenvolver dois sistemas completos: um site de compras para o cliente e um site de gerenciamento para o administrador. Ambos foram construídos com foco na integração entre frontend, backend e banco de dados, além do uso constante de Inteligência Artificial como apoio nas etapas de planejamento, programação, organização e testes.
Durante o desenvolvimento, utilizei a IA tanto para auxiliar na parte visual quanto na lógica interna, buscando entregar um produto funcional, estruturado e compreensível.

2. Descrição do Projeto

O projeto foi dividido em duas partes principais:

Gerenciamento: referente ao projeto do terceiro bimestre, voltado para a criação dos CRUIDs (cadastro, leitura, atualização e exclusão) e toda a área administrativa.

Loja: baseado no projeto do segundo bimestre, mas agora integrado a um banco de dados real, permitindo conexão direta com o sistema de gerenciamento.

A integração entre os dois tornou possível controlar produtos, atualizar informações e visualizar dados de forma dinâmica.

3. O que foi desenvolvido

Durante o projeto, implementei:

Interface visual completa utilizando HTML, CSS e JavaScript no frontend.

Funções de processamento, regras e lógica interna no backend.

Comunicação eficiente entre frontend e backend, garantindo que dados fossem enviados e recebidos corretamente.

Organização do código em arquivos separados, seguindo boas práticas e facilitando manutenção.

Uso de recursos multimídia, como imagens, menus, ícones, botões e mensagens de alerta.

Testes manuais frequentes, com correções e otimizações, para melhorar a experiência do usuário.

Integração com banco de dados, permitindo que informações fossem salvas, carregadas e editadas.

4. Dificuldades encontradas e como foram superadas

Durante o desenvolvimento enfrentei alguns desafios significativos:

4.1 Comunicação com a IA

No início, tive dificuldade em coordenar a IA para que ela seguisse exatamente o que eu queria. Esse problema ficou mais evidente no terceiro bimestre, quando precisei seguir uma estrutura específica para o projeto.
A solução veio quando aprendi a criar prompts melhores — mais claros, mais organizados e mais diretos. Aprendi não só o que pedir, mas como pedir.

4.2 Comunicação entre frontend e backend

Fazer o frontend chamar corretamente o backend foi um dos maiores desafios. Às vezes as funções não respondiam ou os dados não eram enviados de forma adequada.
Para superar isso, reestruturei partes do código, fiz verificações mais detalhadas com ajuda da IA e realizei diversos testes até obter um funcionamento estável.

4.3 Erros lógicos e ajustes no JavaScript

Algumas funcionalidades apresentaram falhas, principalmente nas validações e cálculos. Para resolver, isolei funções, testei em partes menores e utilizei a IA para identificar e corrigir erros mais complexos.

4.4 Layout e responsividade

Organizar o layout também exigiu bastante tempo. Ajustei margens, tamanhos, grids e referências visuais até chegar a um resultado mais limpo e agradável de usar.

5. Experiências com o uso de IA

A Inteligência Artificial foi fundamental para o andamento do projeto. Com ela, consegui:

Tirar dúvidas rapidamente.

Gerar e revisar códigos.

Reescrever partes complexas para maior clareza.

Criar prompts melhores e mais objetivos.

Depurar erros difíceis de localizar sozinho.

O mais importante foi perceber que a IA não serve apenas para entregar código pronto, mas para explicar, ensinar e ajudar a raciocinar sobre o problema. Sem entendimento, qualquer prompt parece grande, mas fica vazio.

6. Expectativas vs. Resultado Final

No começo, eu não esperava conseguir desenvolver um site realmente funcional ainda este ano. Apesar de ainda existirem melhorias, atualizações e correções que gostaria de implementar, estou muito satisfeito com o resultado atual do meu código.
Consegui entregar algo que funciona, é organizado e, principalmente, representa uma evolução enorme em comparação ao início do ano.

7. Preferência entre Backend e Frontend

Não tenho uma preferência absoluta entre backend e frontend, pois acredito que um depende completamente do outro.
O backend é como o cérebro do sistema, enquanto o frontend é o corpo que permite a interação. Mesmo assim, tenho preferência por:

Backend: quando é bem estruturado e organizado.

Frontend: quando é funcional em primeiro lugar e visualmente agradável em segundo.

8. Conclusão

O projeto foi essencial para desenvolver habilidades técnicas e organizacionais. Aprendi a organizar pastas, separar funções, melhorar minha comunicação com a IA e corrigir erros por conta própria.
Também compreendi de forma mais clara como frontend, backend e banco de dados se conectam para formar um sistema real.
No geral, ganhei experiência prática, amadureci como programador e aprendi uma forma muito mais eficiente de estruturar meus códigos e meus projetos.
